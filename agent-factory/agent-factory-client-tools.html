Anti-pattern reference and engagement customization checklist for client delivery of the Agent Factory blueprint.

8 Anti-Patterns This Architecture Prevents

Expand each card to see: the failure mode, how to detect it, and the exact architectural decision that prevents it.

🧱
Monolithic AI
One generalist agent doing everything poorly
⌄
Why It Fails
Prompt bloat causes hallucinations on edge cases. Single point of failure — one bad output poisons trust across all tasks. Context window exhaustion degrades quality on complex tasks. Cannot be tuned per task domain.
⚡ Specialization beats generality
Symptoms to Watch For
Inconsistent quality across task types
SNR varies wildly (80%+ some days, 40% others)
Engineers don't trust output without re-reading
Single agent's failure blocks all automation
How Blueprint Prevents It
15 specialized agents, each with one job
Each agent has a single Markdown definition with narrow, explicit scope. When a new task category emerges, a new agent is created — never expanding an existing one.
🔓
Permission Creep
Agents accumulate permissions over time
⌄
Why It Fails
One hallucination or bad output with write access can corrupt the codebase. Security posture degrades silently — teams notice only after an incident. Violates the principle of least privilege that governs all enterprise security.
🔒 Guardrails enable innovation
Symptoms to Watch For
Observer agents with contents:write
Agents granted admin scope "just in case"
No audit trail showing who granted what
>6 months since last permission review
How Blueprint Prevents It
Hard permission ledger — 15-agent matrix
Semi-annual permission audit required
Three permission tiers (Observer / Proposer / Meta) with explicit YAML templates. Any deviation from least-privilege requires documented justification.
📢
Output Fatigue
Teams stop reading agent outputs because too many are low quality
⌄
Why It Fails
When agent noise exceeds signal, engineers learn to ignore all agent outputs — including the important ones. Trust, once lost, is nearly impossible to rebuild. The factory becomes a spam factory.
📊 Meta-agents are mandatory at scale
Symptoms to Watch For
Agent PRs sit unreviewed for >7 days
Low emoji reaction rates on agent comments
Engineers complain about notification volume
SNR falls below 70% for 2+ weeks
How Blueprint Prevents It
Agent-15: Output Quality Auditor (weekly)
SNR thresholds enforced: alert at <70%
Meta-agents triage all agent outputs and escalate only anomalous findings. New agent deployments are paused if factory SNR drops below threshold.
⬛
Black Box Logic
Agent behavior not inspectable by the team
⌄
Why It Fails
Teams can't trust automation they can't read. When something goes wrong, there's no source of truth to debug. Engineers can't remix or improve agent logic. Democratization of agent creation is impossible.
📄 Markdown as source of truth
Symptoms to Watch For
Agent logic lives only in the LLM vendor UI
Can't explain why an agent made a decision
No audit log of agent actions
Only one person understands how agents work
How Blueprint Prevents It
Markdown DSL — human-readable agent definitions
Full audit logs via GitHub Actions
Every agent's logic lives in `.github/agent-factory/agents/` as Markdown. Any engineer can read, audit, or fork an agent without ML expertise.
✏️
Direct Commits
Agents writing to main branch without human review
⌄
Why It Fails
Removes the human as final authority on code integration. One hallucinated change can corrupt production code. No diff review means no learning signal — teams can't improve agent quality. Violates standard code review culture.
👤 Human-in-the-loop via PRs
Symptoms to Watch For
Agents with direct push to main
No branch protection on key branches
Agent changes bypassing CODEOWNERS
Commits with no human review in history
How Blueprint Prevents It
All Tier 2+ agents create branches + PRs only
No agent granted direct push to main
Tier 2 agents create a branch, commit changes, and open a draft PR with full rationale. Humans review and merge. This is non-negotiable in the permission model.
💸
Cost Spiral
API bills growing without proportional value
⌄
Why It Fails
Longer analysis is not always better — there are real diminishing returns. Defaulting to the most capable (most expensive) model for every task destroys ROI. Without per-agent budgets, cost attribution is impossible.
📉 Match compute energy to complexity
Symptoms to Watch For
All agents using deepest/most expensive model
No per-agent cost tracking
Monthly API bill growing >20%/month
Can't attribute spend to specific agents
How Blueprint Prevents It
Per-agent monthly cost budgets defined
Agent-14: Health Monitor tracks API cost
Three model tiers: Fast (classification/labeling), Standard (analysis/generation), Deep (multi-phase only). Each agent assigned the minimum viable tier for its task.
🌿
Automation Sprawl
Disconnected agents generating noise with no documented purpose
⌄
Why It Fails
Agents created ad hoc for one-off tasks proliferate without governance. No one knows what each agent does or why it exists. Duplication of effort, conflicting outputs, and unaccountable spend become the norm.
📚 Living library as institutional memory
Symptoms to Watch For
Agents not documented in living library
Duplicate agents solving same problem
Team can't list all active agents
No friction-point mapping for any agent
How Blueprint Prevents It
Every agent maps to a documented friction point
Admission gate blocks undocumented agents
The living library at `.github/agent-factory/` is the mandatory registry. The governance checklist requires a documented friction point before any agent can be deployed.
🧟
Stale Agents
Agents running on outdated logic long after their context has changed
⌄
Why It Fails
Codebase evolves; agent Markdown definitions don't. Agents provide accurate-sounding but wrong analysis because they're working from stale assumptions. Teams lose trust as stale agents produce confident-but-incorrect outputs.
🔄 Quarterly deprecation reviews
Symptoms to Watch For
Agent hasn't triggered in 30+ days
Markdown definition not updated in 6+ months
Agent references deprecated patterns/files
Low merge rate on agent PRs (<40%)
How Blueprint Prevents It
30-day inactivity alert via Agent-14
Quarterly deprecation review (mandatory)
Agent-15 SNR scoring catches degradation
Agents with SNR <70% for 2+ quarters are automatically flagged for retirement and moved to `.github/agent-factory/archive/`.

Signal-to-Noise Ratio Quick Reference

The factory's single most important health metric. If SNR drops below target, follow the action immediately.

90–100%
Healthy
Keep deploying
80–90%
Watch Closely
Tune before adding more
70–80%
Pause Deployments
Fix existing first
<70%
Red Alert
Meta-agent audit required
