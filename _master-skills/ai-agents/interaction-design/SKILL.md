# Interaction Design

## Overview

Interaction design (IxD) defines how users interact with digital products and how products respond to user input. This skill focuses on the behavioral layer of design—the micro-interactions, animations, feedback loops, and gesture patterns that make interfaces feel alive and responsive.

Effective interaction design creates products that feel intuitive, responsive, and delightful. It bridges the gap between static UI design and the living, breathing experience users actually engage with.

### Core Competencies

- Micro-interaction design
- Motion and animation principles
- Feedback system design
- Gesture pattern design
- State transition design
- Error and success handling
- Loading and progress indication
- Haptic and audio feedback

## When to Use

### Primary Applications

1. **Micro-Interactions**
   - Button interactions
   - Form feedback
   - Navigation transitions
   - Toggle behaviors

2. **System Feedback**
   - Loading states
   - Progress indicators
   - Success confirmations
   - Error messaging

3. **Motion Design**
   - Page transitions
   - Component animations
   - Scroll-triggered effects
   - Reveal animations

4. **Gesture Design**
   - Touch gestures
   - Swipe behaviors
   - Drag and drop
   - Multi-touch interactions

### Trigger Conditions

- New feature needs interaction design
- Static designs need behavioral layer
- User feedback indicates confusion
- Delight and polish needed
- Mobile gesture patterns required
- Accessibility improvements needed

## Common Pitfalls

### Design Mistakes

| Pitfall | Problem | Prevention |
|---------|---------|------------|
| Over-animation | Distracting, slow | Purposeful motion |
| Inconsistent timing | Jarring experience | Timing system |
| Ignoring accessibility | Excluded users | Reduced motion support |
| Decorative only | No functional value | Purpose-driven |

### Technical Issues

| Pitfall | Problem | Prevention |
|---------|---------|------------|
| Performance impact | Janky experience | 60fps target |
| No fallbacks | Broken experience | Graceful degradation |
| Blocking animations | Frustrated users | Interruptible design |
| Complex implementation | Developer burden | Feasible design |

### Process Problems

| Pitfall | Problem | Prevention |
|---------|---------|------------|
| Late addition | Bolted-on feel | Early integration |
| Vague specs | Inconsistent implementation | Detailed documentation |
| No prototyping | Hard to evaluate | Always prototype |
| Skipping testing | Usability issues | Test with users |

### Timing Guidelines

```
TIMING REFERENCE
├── Instant: 0-100ms (immediate feedback)
├── Fast: 100-200ms (micro-interactions)
├── Normal: 200-300ms (transitions)
├── Slow: 300-500ms (complex animations)
└── Decorative: 500ms+ (ambient motion)
```

### Easing Reference

```
EASING CURVES
├── Ease-out: Entering elements (fast start, slow end)
├── Ease-in: Exiting elements (slow start, fast end)
├── Ease-in-out: On-screen movement (slow-fast-slow)
├── Spring: Playful, bouncy elements
└── Linear: Progress indicators only
```
