# Technical Writing

## Overview

Technical writing transforms complex information into clear, accessible documentation that helps users understand and use products, services, and systems effectively. This skill covers the complete documentation lifecycle from research through publication and maintenance.

Effective technical writing prioritizes clarity, accuracy, and usability over style. The goal is to help users accomplish their tasks with minimal friction, whether they're installing software, using an API, or troubleshooting a problem.

### Core Competencies

- User documentation
- API and developer documentation
- Tutorial and how-to creation
- Knowledge base architecture
- Information design
- Style guide development
- Documentation systems
- Localization preparation

## When to Use

### Primary Applications

1. **Product Documentation**
   - User manuals
   - Getting started guides
   - Feature documentation
   - Release notes

2. **Developer Documentation**
   - API references
   - SDK documentation
   - Integration guides
   - Code examples

3. **Support Content**
   - Knowledge base articles
   - FAQs
   - Troubleshooting guides
   - Error message documentation

4. **Training Materials**
   - Step-by-step tutorials
   - Video scripts
   - Quick reference guides
   - Certification content

### Trigger Conditions

- New product or feature launch
- User support burden high
- Onboarding friction identified
- Documentation debt accumulating
- API or integration expansion
- Compliance documentation required

## Common Pitfalls

### Writing Mistakes

| Pitfall | Problem | Prevention |
|---------|---------|------------|
| Assumptions | Users lost | Write for beginners |
| Jargon overload | Confusion | Define or eliminate |
| Missing steps | Task failure | Test procedures |
| Wall of text | Skimming | Use structure |

### Structural Issues

| Pitfall | Problem | Prevention |
|---------|---------|------------|
| Poor navigation | Content unfindable | Clear IA |
| No search | Discovery failure | Implement search |
| Outdated content | Misinformation | Maintenance plan |
| No versioning | Confusion | Version properly |

### Process Problems

| Pitfall | Problem | Prevention |
|---------|---------|------------|
| No SME access | Inaccuracy | Establish relationships |
| Writing last | Rushing | Parallel development |
| No user testing | Usability issues | Test with users |
| Single author | Knowledge silos | Cross-training |
