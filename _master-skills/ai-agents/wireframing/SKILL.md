# Wireframing

## Overview

Wireframing creates structural blueprints for digital interfaces, focusing on layout, content hierarchy, and functionality before visual design is applied. This skill bridges information architecture and visual design, translating user requirements and content strategy into tangible screen layouts.

Effective wireframes communicate clearly without the distraction of visual styling, enabling rapid iteration and stakeholder alignment on structure and functionality before the investment in detailed design.

### Core Competencies

- Layout and composition
- Content hierarchy
- User flow design
- Interaction patterns
- Responsive wireframing
- Prototype creation
- Annotation and documentation
- Rapid iteration

## When to Use

### Primary Applications

1. **Page Layout**
   - Landing pages
   - Content pages
   - Product pages
   - Dashboard layouts

2. **User Flows**
   - Task flows
   - Checkout processes
   - Onboarding sequences
   - Multi-step forms

3. **Feature Design**
   - New feature concepts
   - Feature iterations
   - Component exploration
   - Interaction patterns

4. **Responsive Design**
   - Desktop layouts
   - Tablet adaptations
   - Mobile layouts
   - Breakpoint planning

### Trigger Conditions

- New page or feature needed
- Layout options to explore
- Stakeholder alignment required
- Content priority decisions
- User flow validation needed
- Before high-fidelity design

## Common Pitfalls

### Design Mistakes

| Pitfall | Problem | Prevention |
|---------|---------|------------|
| Too detailed | Slows iteration | Stay low-fi |
| Too vague | Unclear intent | Sufficient detail |
| Ignoring content | Layout doesn't fit | Real content |
| Desktop only | Mobile problems | Responsive from start |

### Process Issues

| Pitfall | Problem | Prevention |
|---------|---------|------------|
| Skipping sketches | Limited exploration | Always sketch first |
| No user testing | Untested assumptions | Test prototypes |
| Perfect wireframes | Slow progress | Done is better |
| No annotations | Unclear decisions | Document rationale |

### Communication Problems

| Pitfall | Problem | Prevention |
|---------|---------|------------|
| Stakeholder confusion | Wireframe as final design | Set expectations |
| Missing states | Incomplete handoff | Document all states |
| No flow context | Isolated screens | Show connections |
| Poor handoff | Design deviation | Clear documentation |
