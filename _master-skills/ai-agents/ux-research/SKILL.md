# UX Research

## Overview

UX research uncovers user needs, behaviors, and motivations through structured investigation, providing the evidence base for design and product decisions. This skill encompasses the full research lifecycle from planning and data collection through analysis and actionable recommendations.

Effective UX research transforms assumptions into evidence, giving teams confidence that they're building the right things in the right way. This skill covers both qualitative and quantitative methods across discovery, evaluative, and continuous research contexts.

### Core Competencies

- Research planning and methodology
- Interview and moderation skills
- Survey design and analysis
- Observational research
- Data synthesis and analysis
- Insight generation
- Research reporting
- Stakeholder communication

## When to Use

### Primary Applications

1. **Discovery Research**
   - Problem space exploration
   - User needs identification
   - Market opportunity validation
   - Persona development

2. **Evaluative Research**
   - Usability testing
   - Concept validation
   - Design evaluation
   - Competitive analysis

3. **Continuous Research**
   - Ongoing user feedback
   - Feature validation
   - Experience monitoring
   - Trend identification

4. **Strategic Research**
   - Customer segmentation
   - Journey mapping
   - Service design research
   - Longitudinal studies

### Trigger Conditions

- New product or feature exploration
- Design decisions need validation
- User problems poorly understood
- Conversion or engagement issues
- Stakeholder assumptions need testing
- Persona or journey updates needed

## Common Pitfalls

### Research Mistakes

| Pitfall | Problem | Prevention |
|---------|---------|------------|
| Leading questions | Biased data | Neutral wording |
| Small sample | Unreliable insights | Adequate sample |
| Wrong participants | Irrelevant findings | Careful screening |
| Confirmation bias | Missed insights | Diverse analysis |

### Process Issues

| Pitfall | Problem | Prevention |
|---------|---------|------------|
| No clear questions | Unfocused research | Define objectives |
| Rushing analysis | Shallow insights | Allow analysis time |
| Late involvement | Post-hoc validation | Early research |
| Poor documentation | Lost insights | Document everything |

### Activation Problems

| Pitfall | Problem | Prevention |
|---------|---------|------------|
| Report and run | No impact | Active follow-up |
| Too academic | Inaccessible | Practical framing |
| No recommendations | Unclear action | Specific suggestions |
| Siloed insights | Knowledge loss | Central repository |
