# Information Architecture

## Overview

Information architecture (IA) organizes, structures, and labels content to help users find information and complete tasks effectively. This skill creates the structural design of shared information environments, from websites and apps to intranets and knowledge bases.

Effective IA is invisible when done well—users simply find what they need without friction. This skill combines user understanding, content analysis, and structural thinking to create intuitive navigation systems.

### Core Competencies

- Content inventory and audit
- Organizational schemes
- Navigation system design
- Taxonomy and labeling
- Search design
- Site mapping
- User flow design
- IA validation

## When to Use

### Primary Applications

1. **Website/App Structure**
   - New site architecture
   - Site redesign
   - Platform migration
   - Content consolidation

2. **Navigation Design**
   - Primary navigation
   - Secondary navigation
   - Footer navigation
   - Mobile navigation

3. **Taxonomy Development**
   - Category systems
   - Tagging structures
   - Metadata frameworks
   - Faceted classification

4. **Knowledge Management**
   - Intranet organization
   - Documentation structure
   - Help center architecture
   - Resource libraries

### Trigger Conditions

- New digital product development
- Content has grown unwieldy
- Users can't find content
- Navigation is confusing
- Multiple systems need integration
- Content migration planned

## Common Pitfalls

### Structure Mistakes

| Pitfall | Problem | Prevention |
|---------|---------|------------|
| Too deep | Hidden content | Flatten hierarchy |
| Too broad | Overwhelming choice | Group logically |
| Org-chart structure | User mismatch | User-centered design |
| Jargon labels | Confusion | Test with users |

### Navigation Issues

| Pitfall | Problem | Prevention |
|---------|---------|------------|
| Too many items | Cognitive overload | 7 +/- 2 items |
| Inconsistent labeling | Confusion | Label style guide |
| Hidden navigation | Can't find nav | Visible persistence |
| Mobile afterthought | Poor mobile UX | Mobile-first design |

### Process Problems

| Pitfall | Problem | Prevention |
|---------|---------|------------|
| Skipping research | Assumptions | User research first |
| No validation | Untested structure | Card sort + tree test |
| Politics-driven | User ignored | Evidence-based decisions |
| No governance | Drift over time | Maintenance plan |
