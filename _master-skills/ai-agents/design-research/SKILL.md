# Design Research Skill

## Core Workflow


> Master competitive design analysis with trend identification, benchmarking methodologies, and actionable insights for creative strategy.

## Skill Overview

This skill provides expertise in researching and analyzing design landscapes to inform creative decisions. It covers competitive analysis, trend identification, and design benchmarking.

## Core Capabilities

### Competitive Analysis
- Visual audit methodologies
- Feature comparison
- Positioning analysis
- Experience mapping
- Strength/weakness assessment
- Opportunity identification

### Trend Research
- Trend monitoring
- Pattern recognition
- Industry tracking
- Cross-category analysis
- Forecast modeling
- Validation methods

### Benchmarking
- Best-in-class identification
- Metric comparison
- Experience evaluation
- Standard setting
- Gap analysis
- Improvement planning

## Competitive Design Analysis

### Analysis Framework
```yaml
competitive_analysis_structure:
  scope_definition:
    direct_competitors:
      definition: "Same product/service, same market"
      typically: "3-5 companies"

    indirect_competitors:
      definition: "Different product, same need"
      typically: "2-3 companies"

    aspirational:
      definition: "Best-in-class regardless of category"
      typically: "2-3 brands"

  analysis_dimensions:
    visual_identity:
      - Logo and brand marks
      - Color palette
      - Typography
      - Imagery style
      - Iconography

    digital_presence:
      - Website design
      - Mobile experience
      - App design
      - Social media
      - Email design

    content_strategy:
      - Messaging hierarchy
      - Tone of voice
      - Content types
      - Value proposition

    user_experience:
      - Navigation patterns
      - Interaction design
      - Conversion flows
      - Accessibility
```

## Benchmarking

### Benchmarking Framework
```yaml
benchmarking_approach:
  types:
    competitive_benchmark:
      purpose: "Compare to direct competitors"
      focus: "Market position, parity features"

    best_in_class:
      purpose: "Learn from leaders"
      focus: "Excellence standards, aspiration"

    functional:
      purpose: "Compare specific functions"
      focus: "Deep dive on one area"

  process:
    step_1: "Define what to benchmark"
    step_2: "Identify benchmark subjects"
    step_3: "Collect data systematically"
    step_4: "Analyze gaps and strengths"
    step_5: "Set improvement targets"
    step_6: "Develop action plan"
```

### UX Benchmarking
```yaml
ux_benchmark_metrics:
  usability:
    task_success_rate:
      description: "% completing key tasks"

    time_on_task:

      benchmark: "Best-in-class times"

  perception:
    system_usability_scale:
      description: "SUS score (0-100)"
      benchmark: "68+ is above average"

    net_promoter_score:
      description: "Recommendation likelihood"

  engagement:
    bounce_rate:
      benchmark: "Industry average by page type"

    session_duration:
      benchmark: "Category comparison"

    conversion_rate:
      benchmark: "Competitor average"
```
