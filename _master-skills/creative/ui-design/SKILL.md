# UI Design

## Overview

UI (User Interface) design focuses on the visual and interactive elements users engage with in digital products. This skill covers the creation of aesthetically pleasing, functional, and consistent interfaces that align with brand identity while optimizing for usability and accessibility.

Modern UI design extends beyond individual screens to encompass comprehensive design systems—collections of reusable components, patterns, and guidelines that ensure consistency at scale. This skill addresses both the creative and systematic aspects of interface design.

### Core Competencies

- Visual hierarchy and layout design
- Component and pattern design
- Design system architecture
- Typography and color in digital contexts
- Iconography and visual elements
- Responsive and adaptive design
- Accessibility-first design
- Design-to-development handoff

## When to Use

1. **New Product Design**

3. **Product Redesign**

4. **Feature Design**

## Common Pitfalls

### Design Mistakes

| Pitfall | Problem | Prevention |
|---------|---------|------------|
| Inconsistent spacing | Visual chaos | Use spacing tokens |
| Poor contrast | Accessibility failure | Check WCAG early |
| Over-designing | Complexity, slow load | Embrace simplicity |
| Ignoring states | Incomplete experience | Design all states |

### System Issues

| Pitfall | Problem | Prevention |
|---------|---------|------------|
| Too rigid | Stifles innovation | Build in flexibility |
| Too flexible | Inconsistency | Clear governance |
| Under-documented | Misuse and confusion | Thorough docs |
| No versioning | Breaking changes | Semantic versioning |

### Process Problems

| Pitfall | Problem | Prevention |
|---------|---------|------------|
| Siloed design | Dev implementation gaps | Collaborate early |
| No user testing | Assumptions persist | Test with users |
| Ignoring platform | Feels foreign to users | Follow conventions |
| Scope creep | Never ships | Define MVP clearly |
