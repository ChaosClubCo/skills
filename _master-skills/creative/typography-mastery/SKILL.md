# Typography Mastery

> Command the invisible architecture of language — where letterforms become meaning, hierarchy becomes navigation, and type becomes voice.

## Description

This skill covers the art and science of typography, from font selection and pairing through hierarchical systems, readability optimization, and expressive typographic design. It provides frameworks for building type systems that scale across media, selecting typefaces that reinforce brand voice, and applying the principles of contrast, rhythm, and proportion that make text both legible and beautiful. The skill addresses web typography, print typography, display lettering, and the systematic thinking required to maintain typographic consistency across complex design systems.

## Instructions

### Core Workflow

**Step 1: Context and Purpose Analysis**
- Define the medium: screen, print, environmental, packaging, motion
- Identify the audience: age, reading context, cultural expectations, accessibility needs
- Establish the voice: authoritative, friendly, luxurious, technical, playful, urgent
- Determine the content types: long-form reading, headings, data, UI elements, display
- Audit existing typography if redesigning — identify what works and what fails

**Step 2: Typeface Selection**
- Select a primary typeface that embodies the project's voice and personality
- Evaluate optical qualities: x-height, contrast, width, weight range, character set completeness
- Verify technical requirements: variable font support, web licensing, language coverage
- Choose a secondary typeface that provides functional contrast to the primary
- Test both typefaces at all sizes they will be used — a font beautiful at 72pt may fail at 12pt

**Step 3: Hierarchical System Design**
- Establish a modular type scale using a consistent ratio (1.25, 1.333, 1.414, 1.5, or 1.618)
- Define 5-7 hierarchy levels: display, H1, H2, H3, body, caption, overline
- Assign weight, size, tracking, and color to each level
- Set line-height (leading) proportional to font size: 1.2 for headings, 1.5-1.7 for body
- Document the system with named tokens or style definitions

**Step 4: Composition and Layout**
- Set optimal line length (measure): 45-75 characters per line for body text
- Establish a baseline grid to align text across columns and components
- Apply tracking (letter-spacing) adjustments: tighter for large display, looser for small caps and captions
- Design paragraph spacing, indentation, and list styling for content flow
- Create responsive rules for how typography adapts across breakpoints and surfaces

**Step 5: Testing and Refinement**
- Test readability across devices, sizes, and viewing distances
- Evaluate in real content — lorem ipsum hides typographic problems
- Check accessibility: WCAG contrast ratios, minimum font sizes, dyslexia-friendly options
- Render test across browsers and operating systems for web typography
- Document the complete type system for design team adoption

### Type Selection and Pairing Framework

Choose and combine typefaces using structural contrast and tonal harmony — never pair fonts that are merely different; pair fonts that are purposefully complementary.

**The Anatomy of Selection:**
Every typeface carries embedded meaning before a single word is read. A geometric sans-serif communicates modernity and precision. A humanist serif communicates warmth and tradition. A high-contrast didone communicates elegance and authority. A monospace communicates technicality and code. Selection begins with understanding what the letterforms say silently.

**Evaluating a Typeface:**

1. **x-height:** The height of lowercase letters relative to capitals. Typefaces with a large x-height (like Roboto, Open Sans) are more legible at small sizes. Typefaces with a smaller x-height (like Garamond, Didot) feel more refined but require larger point sizes for readability.

2. **Contrast:** The ratio between thick and thin strokes. High-contrast faces (Bodoni, Playfair Display) are dramatic and elegant but can lose thin strokes at small sizes or low resolution. Low-contrast faces (Futura, Helvetica) are stable and reliable across all sizes.

3. **Width and Proportion:** Condensed faces conserve horizontal space and feel urgent or editorial. Extended faces feel open and contemporary. Standard-width faces are the safest choice for body text.

4. **Weight Range:** A typeface family with many weights (thin through black) provides enormous flexibility within a single design language. Variable fonts offer continuous weight axes for precise control.

5. **Character Set:** Verify that the typeface supports all required characters: accented Latin, Cyrillic, Greek, numeric sets, currency symbols, mathematical symbols. Missing characters create visible inconsistency.

**Pairing Principles:**

- **Contrast, not conflict:** Pair a serif with a sans-serif, a geometric with a humanist, a high-contrast with a low-contrast. The pairing should create visual tension that is interesting, not jarring.
- **Shared skeleton:** The best pairings share underlying proportions — similar x-heights, similar character widths, similar stroke terminals — while differing in classification. They rhyme without repeating.
- **Role clarity:** Each typeface in a pairing must have a clear job. Typically: one for headings (personality, expression) and one for body text (readability, neutrality). Never use both at the same size and weight.
- **The two-typeface rule:** Most projects need exactly two typefaces. Three is a maximum. Four or more creates visual chaos. Variety comes from weight, size, and color — not from adding more fonts.

**Reliable Pairing Strategies:**

- **Superfamily pairing:** Choose a family that includes both serif and sans-serif variants designed to work together (e.g., Source Serif + Source Sans, Merriweather + Open Sans). Guaranteed harmony.
- **Historical period pairing:** Combine typefaces from the same era or movement. They share proportional DNA even if they differ in style.
- **Contrast axis pairing:** Identify the primary axis of contrast (serif vs. sans, geometric vs. humanist, compressed vs. extended) and pair across that single axis while keeping other qualities similar.

### Hierarchy and Readability Framework

Design type hierarchies that guide the reader's eye through content with effortless clarity — making the structure of information visible without being intrusive.

**The Purpose of Hierarchy:**
Typographic hierarchy answers three questions for the reader instantly: What is this? What is most important? Where do I go next? Without clear hierarchy, every piece of text competes for attention equally, and the reader's cognitive load increases dramatically.

**Building Hierarchy Through Contrast:**
There are exactly four tools for creating typographic contrast. Used together, they produce clear, scalable hierarchies.

1. **Size contrast:** The most powerful differentiator. A headline at 48pt and body at 16pt creates a 3:1 ratio that is unmistakably hierarchical. Minimum useful ratio: 1.5:1.

2. **Weight contrast:** Bold versus regular, or black versus light. Weight contrast works within a single typeface family and is essential for UI design where size variation may be constrained.

3. **Color/value contrast:** Dark versus light, or chromatic accent versus neutral. Use sparingly — too many colors fragment the hierarchy rather than clarifying it.

4. **Spatial contrast:** The amount of whitespace around an element signals its importance. Generous margin above a heading separates it from preceding content and gives it visual authority.

**The Modular Scale:**
A modular type scale generates font sizes from a base size multiplied by a consistent ratio.

Base size: 16px (standard browser default)
- Ratio 1.250 (Major Third): 16, 20, 25, 31, 39, 49
- Ratio 1.333 (Perfect Fourth): 16, 21, 28, 38, 50, 67
- Ratio 1.414 (Augmented Fourth): 16, 23, 32, 45, 64, 90
- Ratio 1.618 (Golden Ratio): 16, 26, 42, 68, 109, 177

Choose a tighter ratio (1.125-1.250) for interfaces with many hierarchy levels in constrained space. Choose a wider ratio (1.414-1.618) for editorial designs with dramatic typographic expression.

**Readability Essentials:**

- **Line length (measure):** 45-75 characters per line. Shorter for columns, longer for single-column layouts. Lines too long cause the eye to lose tracking on the return sweep. Lines too short cause excessive hyphenation and choppy reading rhythm.
- **Line height (leading):** Body text requires 1.4-1.7x the font size. Headings can be tighter (1.1-1.3x). Increase leading as line length increases to help the eye track back to the left margin.
- **Paragraph spacing:** Use space-after (typically 0.5-1.0x the body font size) rather than indentation for screen text. For print, first-line indentation (1-2em) is traditional.
- **Tracking (letter-spacing):** Body text is set at the type designer's default spacing. Increase tracking for small text (captions, labels) and uppercase text. Decrease tracking for large display text where letter spacing appears visually loose.

### Best Practices

- Never use more than three typefaces in a single project — constraint produces coherence
- Set body text at a minimum of 16px for screen and 10pt for print — smaller is not more sophisticated, it is unreadable
- Avoid justified text on screen unless the column width supports it without excessive word spacing
- Use true italics, not oblique or faux italic — the difference is visible and meaningful
- Kern display text manually — automatic kerning is optimized for body text, not headlines
- Test typography with real content in the target language — character frequency and word length vary dramatically
- Use tabular (monospaced) figures for data tables and columns, proportional figures for running text
- Apply optical margin alignment (hanging punctuation) for refined print typography
- Never stretch or compress a typeface — if you need a different width, choose a condensed or extended cut
- Increase letter-spacing for uppercase text by 5-10% to improve readability
- Set line-height and paragraph spacing as relative units (em, rem) rather than fixed pixels for scalable systems
- Ensure all heading levels are semantically correct in HTML, not just visually different
- Prefer variable fonts for web projects — one file replaces multiple weight files and enables fluid responsive scaling
- Always check typographic rendering on Windows — it handles anti-aliasing and hinting differently than macOS
- Design the smallest text first — if captions and labels are legible and clear, the larger sizes will follow naturally

### Common Patterns

**Pattern: The SaaS Product Type System**
A software product establishes its typographic system using Inter as the sole typeface, leveraging its extensive weight range (Thin through Black) and variable font capability. The type scale uses a 1.25 ratio with a 14px base for the application interface and a 16px base for marketing pages. Six hierarchy levels are defined: Page Title (28px Bold), Section Heading (22px Semibold), Subsection (18px Medium), Body (14px Regular), Label (12px Medium, uppercase, +0.5px tracking), and Help Text (12px Regular, secondary color). Line heights are set at 1.2 for headings and 1.5 for body. The system is tokenized in the design system with semantic naming: --type-display, --type-heading, --type-body, --type-caption. All text meets WCAG AA contrast requirements against both light and dark backgrounds.

**Pattern: The Editorial Magazine Redesign**
A print magazine redesigns its typography to pair Lyon Display (a high-contrast serif) for headlines with Atlas Grotesk (a humanist sans-serif) for body text. The headline typeface brings editorial authority and drama with its sharp serifs and elegant contrast. The body typeface provides quiet, readable companionship at 9.5pt with 12pt leading across a 3-column grid with 28-pica column widths (approximately 55 characters per line). Pull quotes use Lyon Display Italic at 24pt with generous leading. Department headings use Atlas Grotesk Bold in small caps with 80-unit tracking. Captions use Atlas Grotesk Regular at 7.5pt. The system maintains exactly two typefaces across 120 pages, achieving variety exclusively through weight, size, style, and spatial treatment.

**Pattern: The Responsive Web Typography Scale**
A content-heavy website implements a fluid type scale that adapts continuously between breakpoints rather than jumping at fixed points. The base font size scales from 16px on mobile to 18px on desktop using CSS clamp(). The heading scale ratio shifts from 1.2 on mobile (conserving vertical space) to 1.333 on desktop (creating more dramatic hierarchy). Line length is constrained using max-width on the content container: 65ch for articles, 45ch for sidebar content. The type system uses a single variable font (Source Sans 3) loaded as a single WOFF2 file, reducing font loading to one HTTP request while providing access to all weights from ExtraLight to Black. Font-display: swap ensures text remains visible during font loading.

### Output Formats

**Type System Specification:**
A complete, implementation-ready document defining every typographic decision: typeface selections with licensing details, the modular scale with all size/weight/leading/tracking values, color assignments, responsive behavior rules, and accessibility compliance data. Formatted for direct adoption by design and development teams.

**Font Selection Rationale:**
An evaluative document presenting 2-3 typeface candidates with comparative analysis across optical quality, technical capability, licensing, rendering, and brand alignment. Includes specimen settings at all intended sizes and recommended pairings. Supports informed decision-making by stakeholders.

**Typographic Style Guide:**
A usage-focused reference document with visual examples showing correct and incorrect application of the type system across all project surfaces. Includes DO/DON'T examples, edge case guidance, and responsive behavior specifications. Designed for consistent adoption across a design team.
