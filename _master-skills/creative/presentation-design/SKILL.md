# Presentation Design

> Transform ideas into visual narratives that move audiences from attention to action.

## Description

This skill covers the complete discipline of presentation design, from narrative architecture and slide composition to data visualization and delivery support. It provides frameworks for structuring persuasive arguments visually, designing slides that amplify rather than compete with the speaker, and presenting complex data with clarity and impact. The skill addresses visual hierarchy, typographic choices, color systems, animation restraint, and the critical relationship between what is shown and what is spoken. It serves both the designer creating slides and the speaker delivering them.

## Instructions

### Core Workflow

**Step 1: Narrative Architecture**
- Define the single core message the audience must remember (the "throughline")
- Identify the audience: their knowledge level, concerns, decision-making power
- Choose a narrative structure: problem-solution, journey, comparison, revelation
- Outline the presentation arc: opening hook, building argument, climax, resolution, call to action
- Determine the emotional journey — where should the audience feel tension, relief, excitement, urgency

**Step 2: Content Hierarchy and Slide Planning**
- Map the outline to slides: one idea per slide, no exceptions
- Establish the content hierarchy for each slide: headline, supporting point, evidence
- Decide what lives on the slide versus what lives in the speaker's voice
- Plan transition logic — each slide should answer "why does this follow what came before?"
- Create a slide-by-slide storyboard before touching any design tool

**Step 3: Visual Composition**
- Select a typographic system: one sans-serif for headlines, one for body, consistent sizing scale
- Establish a color palette: 1 primary, 1 accent, 2 neutrals maximum
- Design layouts using a grid system for consistent alignment across all slides
- Apply visual hierarchy through size contrast, weight contrast, and spatial positioning
- Use whitespace aggressively — empty space directs attention to what remains

**Step 4: Data Visualization and Evidence**
- Choose chart types based on the relationship being shown, not the data shape
- Simplify every chart to its essential message — remove gridlines, reduce labels, highlight the key data point
- Use annotation to guide the audience's interpretation of data
- Integrate data into the narrative — every number must connect to a human impact
- Design comparison slides that make the preferred option visually obvious

**Step 5: Refinement and Delivery Support**
- Review every slide at presentation distance — if you cannot read it from 10 feet, simplify
- Prepare speaker notes with talking points, transition phrases, and timing cues
- Build a narrative bridge document connecting slide content to spoken narrative
- Rehearse with the slides to identify pacing issues and visual-verbal mismatches
- Export in appropriate formats: widescreen 16:9, PDF backup, print handouts

### Visual Storytelling Framework

Design slides as visual arguments where every element serves the narrative.

**The Slide as a Billboard:**
Each slide should function like a highway billboard — communicable in 3 seconds or less. If the audience is reading your slide, they are not listening to you. The slide supports the speaker; the speaker does not narrate the slide.

**Headline-Driven Design:**
Every slide needs a headline that states a conclusion, not a topic. "Q3 Revenue" is a topic. "Q3 Revenue Grew 34% Despite Market Contraction" is a conclusion. The headline tells the audience what to think; the visual evidence shows them why.

**The Three Composition Principles:**

1. **Contrast Creates Hierarchy.** The most important element must be visually dominant. Use size, color, weight, and position to create unmistakable focal points. If everything is bold, nothing is bold.

2. **Alignment Creates Order.** Every element on the slide should align to a grid or to other elements. Misalignment — even by a few pixels — creates unconscious visual noise that erodes credibility.

3. **Proximity Creates Meaning.** Elements placed close together are perceived as related. Use spatial grouping to show relationships and generous spacing to show separation. White space is not empty — it is structural.

**Slide Type Taxonomy:**
- **Title slides:** Bold, minimal, atmospheric. Set the tone for what follows.
- **Statement slides:** One sentence, large typography, centered. Maximum impact for key claims.
- **Evidence slides:** Chart, image, or quote that supports the preceding statement. Annotated and simplified.
- **Framework slides:** Diagrams showing relationships, processes, or models. Clean lines, consistent shapes, clear labels.
- **Transition slides:** Brief section dividers that reorient the audience. Often a single word or question.
- **Summary slides:** Structured recaps using consistent formatting. Never more than 5 items.

**Animation Philosophy:**
Animation should reveal, not decorate. Use builds to control information flow — showing one point at a time prevents the audience from reading ahead. Transitions between slides should be invisible (simple cuts or fades). Avoid fly-ins, spins, bounces, or any motion that draws attention to itself rather than to the content.

### Data Presentation Framework

Present data as stories with characters, conflict, and resolution — not as spreadsheets projected on walls.

**Chart Selection by Message:**
The chart type must match the story the data tells.

- **Comparison:** Bar charts (vertical for few categories, horizontal for many or long labels)
- **Change over time:** Line charts (continuous data), slope charts (two time points)
- **Composition:** Stacked bar charts, treemaps (not pie charts — they are nearly always inferior)
- **Distribution:** Histograms, box plots, scatter plots
- **Relationship:** Scatter plots, bubble charts
- **Geographic:** Choropleth maps, proportional symbol maps

**The Data Storytelling Process:**

1. **Find the story.** Before designing, identify the single insight the data reveals. What is surprising, concerning, or encouraging? That is your story.

2. **Simplify ruthlessly.** Remove all chart elements that do not serve the story. Default chart formatting from tools like Excel is designed for analysis, not presentation. Strip gridlines, reduce axis labels, remove legends when direct labeling is possible.

3. **Annotate the insight.** Add a text callout that explicitly states what the audience should see. Circle the anomaly. Draw the trend line. Label the gap. Never assume the audience will find the insight on their own.

4. **Connect to consequence.** Pair every data slide with a "so what" statement. "Revenue grew 34%" means nothing without "...which means we can fund two additional product lines."

**Color in Data:**
- Use a single highlight color to draw attention to the key data point
- Gray out all non-essential data elements to reduce visual competition
- Never use more than 5 colors in a single chart
- Ensure color choices are accessible to colorblind viewers (avoid red-green distinctions)
- Use sequential color scales for ordered data, diverging scales for data with a meaningful midpoint

**Common Data Presentation Mistakes:**
- 3D charts that distort proportions and obscure actual values
- Dual-axis charts that imply false correlations
- Truncated axes that exaggerate small differences
- Overloaded charts that show every data point rather than the relevant story
- Pie charts with more than 3 segments

```

AUDIENCE: [Who they are and what they care about]
DURATION: [Minutes]
CORE MESSAGE: [One sentence the audience must remember]
CALL TO ACTION: [What you want the audience to do after]

OPENING HOOK (Slides 1-3):
  Slide 1: [Title slide — topic, speaker, context]
  Slide 2: [Hook — surprising stat, provocative question, or story]
  Slide 3: [Agenda/roadmap — what you will cover and why it matters]

ACT ONE — THE PROBLEM (Slides 4-8):
  Slide 4: [Statement of the problem]
  Slide 5: [Evidence — data showing scale or urgency]
  Slide 6: [Human impact — story or example]
  Slide 7: [Why existing solutions fail]
  Slide 8: [Transition — "What if there were a better way?"]

ACT TWO — THE SOLUTION (Slides 9-15):
  Slide 9: [Solution overview — one sentence]
  Slide 10: [How it works — framework or process]
  Slide 11: [Evidence — data showing effectiveness]
  Slide 12: [Case study or proof point 1]
  Slide 13: [Case study or proof point 2]
  Slide 14: [Comparison — before vs. after]
  Slide 15: [Transition — "Here's what this means for you"]

ACT THREE — THE IMPACT (Slides 16-20):
  Slide 16: [Projected outcomes]
  Slide 17: [Implementation roadmap]
  Slide 18: [Investment and timeline]
  Slide 19: [Summary — 3 key takeaways]
  Slide 20: [Call to action + contact]

SPEAKER NOTES PER SLIDE:
  [Talking points, timing target, transition phrase to next slide]
```

### Best Practices

- One idea per slide — if you need a bullet list, you need more slides
- Design at the back of the room — if it is not legible from 30 feet, the text is too small
- Use full-bleed photography to create emotional impact on key narrative moments
- Never read your slides aloud — the slide shows, the speaker tells
- Limit text to 30 words per slide maximum; aim for fewer than 15
- Use consistent slide transitions — invisible cuts signal professionalism
- Left-align text by default; centered text only for single statements
- Present data progressively — show the axes first, then the data, then the annotation
- Use speaker notes extensively — they are the presentation; slides are the scenery
- Build redundancy: carry your slides on USB, email them to yourself, have a PDF backup
- Rehearse with a timer to ensure you stay within your time allocation
- Design a "leave-behind" version with additional detail for post-presentation distribution
- End with a clear, specific call to action — not "any questions?" but "here is what I need from you"
- Use high-quality images only — one pixelated photo undermines the entire presentation's credibility
- Test your slides on the actual presentation display before the audience arrives

### Common Patterns

**Pattern: The Investor Pitch Deck**
A startup prepares a 12-slide Series A pitch. The narrative follows problem-solution-traction-ask. The opening slide uses a single provocative statistic about market size. The problem slides use customer quotes in large typography — real voices, not abstractions. The solution slide shows the product in context with a clean screenshot and three benefit callouts. The traction slide uses a single line chart with revenue growth annotated at key milestones. The team slide uses professional photos with one-line bios emphasizing relevant experience. The ask slide states the specific funding amount, use of funds in three categories, and projected 18-month outcomes. Every slide follows a headline-driven design with conclusions, not topics.

**Pattern: The Conference Keynote**
A designer prepares a 30-minute keynote for a 500-person conference. The presentation uses 80 slides — an average of one every 22 seconds. Most are single-image or single-statement slides that create rhythm and momentum. The talk opens with a personal story spanning 8 slides that establishes emotional connection. Three framework slides use simple diagrams built progressively with animations. Data slides use a single highlighted data point against a grayed-out context. The closing sequence revisits the opening story with new meaning, creating a circular narrative that lands the core message. Speaker notes contain full scripted transitions between sections.

**Pattern: The Executive Status Update**
A product manager presents quarterly results to the leadership team. The 15-minute presentation uses 10 slides. An executive summary slide presents the three key metrics with red/yellow/green status indicators and one-sentence explanations. The following slides each deep-dive into one metric with a chart showing trend over four quarters, annotated with the specific actions that drove results. A risks-and-mitigations slide uses a simple two-column layout. The final slide presents three prioritized asks from leadership, each with a clear deadline and owner. The entire deck uses the company's brand typography and color system for instant visual credibility.

### Output Formats

**Presentation Outline and Storyboard:**
A structured narrative document containing the throughline, audience analysis, slide-by-slide content plan with headlines, visual direction notes, speaker notes, and timing allocations. Ready for a designer to execute or a speaker to build.

**Slide Design System:**
A specification document defining the visual rules for the presentation: typography scale, color palette, grid system, chart styles, image treatment, animation rules, and master slide layouts. Ensures visual consistency across any number of contributors.

**Speaker Support Package:**
A delivery preparation document containing the full speaker script or talking points per slide, transition phrases, timing targets, audience interaction moments, Q&A preparation, and technical setup requirements for the venue.
