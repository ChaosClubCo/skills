# Budgeting & Financial Planning

## Overview

Budgeting and financial planning translate business strategy into financial targets and resource allocation. This skill covers the complete planning cycle from annual budget development through ongoing forecasting and variance management. Effective budgeting enables SMBs to set financial goals, allocate resources strategically, control spending, measure performance, and make informed business decisions.

For SMB business solutions, disciplined budgeting processes provide financial guardrails, enable growth planning, support investor/lender requirements, and create accountability across the organization.

## When to Use

Invoke this skill when:
- Developing annual operating budgets
- Creating department or project budgets
- Allocating resources across business units
- Analyzing budget vs actual variance
- Building rolling forecasts
- Conducting scenario planning for strategic decisions
- Planning capital expenditures
- Preparing budget presentations for leadership/board
- Setting up budget tracking and reporting
- Making mid-year budget adjustments
- Building financial models for new initiatives

### 2. Department Budget Allocation

#### Budget Allocation Methods

```
ALLOCATION METHODOLOGIES

INCREMENTAL BUDGETING
Prior year + adjustment %
- Simple and quick
- Good for stable operations
- May perpetuate inefficiencies

ZERO-BASED BUDGETING (ZBB)
Build from scratch, justify all spending
- Eliminates waste
- Time-intensive
- Best for cost reduction focus

ACTIVITY-BASED BUDGETING (ABB)
Budget based on activity drivers
- Links costs to outputs
- Requires activity analysis
- Good for service departments

DRIVER-BASED BUDGETING
Key metrics drive budget
- Revenue drives S&M spend
- Headcount drives overhead
- Good for variable costs

HYBRID APPROACH (RECOMMENDED)
- ZBB for discretionary (every 3 years)
- Driver-based for variable
- Incremental for fixed
- Activity-based for shared services
```

#### Variance Classification

```
VARIANCE TYPE FRAMEWORK

VOLUME VARIANCE
- More/fewer units than planned
- Customer count differs
- Transaction volume changed
- Formula: (Actual Units - Budget Units) x Budget Price

PRICE VARIANCE
- Selling price differs from budget
- Cost per unit changed
- Rate variance
- Formula: (Actual Price - Budget Price) x Actual Units

MIX VARIANCE
- Product mix shifted
- Customer segment changed
- Channel mix different
- Impact: Different margin contribution

TIMING VARIANCE
- Revenue recognized differently
- Expense timing shifted
- Accrual adjustments
- Action: May self-correct

EFFICIENCY VARIANCE
- Productivity differences
- Process improvements
- Resource utilization
- Indicates operational performance

PERMANENT vs TEMPORARY
- Permanent: Adjust forecast going forward
- Temporary: No forecast change needed
- Document classification for each variance
```

### 4. Rolling Forecasts

#### Rolling Forecast Framework

```
ROLLING FORECAST METHODOLOGY

FORECAST HORIZON
- Rolling 4-6 quarters (12-18 months)
- Updated monthly or quarterly
- Always looking forward same distance

FORECAST LEVELS
1. Total company
2. Business unit/segment
3. Department
4. Key line items

UPDATE CYCLE
Monthly Light Touch:
- Update current quarter actuals
- Adjust next 2 quarters for known changes
- Roll forward one month

Quarterly Deep Dive:
- Full re-forecast all periods
- Department input
- Detailed assumption review
- New initiative impacts

### 5. Scenario Planning

#### Scenario Framework

```
SCENARIO PLANNING APPROACH

SCENARIO DEFINITIONS

BASE CASE (Most Likely)
- Current trajectory continues
- Known deals close as expected
- Planned initiatives execute
- No major disruptions

UPSIDE CASE (Best Case)
- Accelerated growth
- Key deals close early
- New opportunities materialize
- Market tailwinds

DOWNSIDE CASE (Conservative)
- Growth slows
- Some deals slip/lost
- Economic headwinds
- Increased competition

STRESS TEST (Worst Case)
- Significant downturn
- Major customer loss
- Market disruption
- Survival scenario

## Common Pitfalls

### Budget Development Errors

1. **Sandbagging**: Budgets too conservative
2. **Hockey stick**: Unrealistic back-end loading
3. **No ownership**: Finance-driven without input
4. **Too detailed**: Can't see forest for trees
5. **Disconnected from strategy**: Budget doesn't align

### Execution Failures

1. **Set and forget**: No ongoing monitoring
2. **Gaming**: Spending to budget, not need
3. **Slow reporting**: Data too stale to act
4. **No consequences**: Variances not addressed
5. **Rigid**: No mechanism for mid-year changes

### Process Issues

1. **Too long**: Budget process is exhausting
2. **Wrong cadence**: Annual-only is outdated
3. **Spreadsheet chaos**: Version control nightmare
4. **No scenario planning**: Single point forecast
5. **Poor communication**: Departments in the dark
