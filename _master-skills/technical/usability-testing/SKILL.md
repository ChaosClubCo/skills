# Usability Testing

## Overview

Usability testing evaluates how well users can complete tasks using a product or prototype, identifying friction points and improvement opportunities through direct observation. This skill covers the complete testing lifecycle from study design through analysis and recommendations.

Usability testing provides direct evidence of user behavior, moving beyond opinion and assumption to understand what users actually do. This skill emphasizes practical, actionable testing that improves products.

### Core Competencies

- Test planning and design
- Task scenario development
- Moderation techniques
- Observation and note-taking
- Analysis and synthesis
- Issue prioritization
- Recommendation development
- Stakeholder communication

## When to Use

### Primary Applications

1. **Prototype Testing**
   - Low-fidelity validation
   - High-fidelity evaluation
   - Interaction testing
   - Flow verification

2. **Live Product Testing**
   - Feature evaluation
   - Redesign validation
   - Competitive comparison
   - Baseline establishment

3. **Comparative Testing**
   - A/B design comparison
   - Before/after studies
   - Competitive benchmarking
   - Design option evaluation

4. **Specialized Testing**
   - Accessibility testing
   - Mobile usability
   - First-use testing
   - Expert reviews

### Trigger Conditions

- New design needs validation
- Conversion or completion issues
- Stakeholder disagreement on design
- Pre-launch validation required
- Design iteration checkpoint
- Competitive comparison needed

## Common Pitfalls

### Test Design Mistakes

| Pitfall | Problem | Prevention |
|---------|---------|------------|
| Leading tasks | Biased results | Neutral wording |
| Too many tasks | Participant fatigue | 5-7 tasks max |
| Unrealistic scenarios | Invalid behavior | Real context |
| Wrong participants | Irrelevant insights | Rigorous screening |

### Moderation Issues

| Pitfall | Problem | Prevention |
|---------|---------|------------|
| Helping too much | Masked problems | Stay neutral |
| Leading questions | Biased answers | Open questions |
| Talking too much | Missed observations | Embrace silence |
| Rigid script | Missed opportunities | Flexible probing |

### Analysis Problems

| Pitfall | Problem | Prevention |
|---------|---------|------------|
| Cherry-picking | Biased conclusions | Systematic review |
| One participant focus | Overweighting | Pattern focus |
| Solution jumping | Missed root cause | Why analysis |
| No prioritization | Overwhelmed team | Severity matrix |
