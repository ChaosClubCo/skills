# Creative Briefs Skill

## Core Workflow


> Master creative brief development with structured templates, stakeholder alignment processes, and comprehensive project scoping methodologies.

## Skill Overview

This skill provides expertise in developing effective creative briefs that align stakeholders, clarify objectives, and set creative projects up for success. It covers brief templates, discovery processes, and communication strategies.

### Brief Development
- Strategic brief writing
- Creative requirements definition
- Objective clarification

- Constraint documentation
- Timeline specification

### Stakeholder Alignment
- Discovery workshops
- Requirement gathering
- Expectation management
- Sign-off processes
- Conflict resolution
- Communication frameworks

### Project Scoping
- Scope definition
- Deliverable specification
- Resource estimation
- Risk identification
- Dependency mapping
- Milestone planning

## Discovery Process

### Stakeholder Interview Guide
```yaml
discovery_questions:
  business_context:
    - "What business problem are we solving?"
    - "How does this fit into broader business goals?"
    - "What does success look like?"
    - "What happens if we don't do this?"

  audience_understanding:
    - "Who exactly are we talking to?"
    - "What do they currently think about us?"
    - "What motivates their decisions?"
    - "Where do we reach them?"

  creative_direction:
    - "What examples do you love? Why?"
    - "What should we absolutely avoid?"
    - "How bold can we be?"
    - "What's non-negotiable?"

  practical_constraints:
    - "What's the real budget?"
    - "What's the hard deadline?"
    - "Who needs to approve?"
    - "What existing assets must we use?"
```

## Brief Quality Checklist

### Completeness Check
```yaml
essential_elements:
  strategic:
    - [ ] Clear business objective defined
    - [ ] Specific communication objective stated
    - [ ] Target audience identified with insight
    - [ ] Single-minded proposition articulated
    - [ ] Success metrics established

  practical:
    - [ ] All deliverables listed with specs
    - [ ] Timeline with key milestones
    - [ ] Budget confirmed
    - [ ] Approval process defined
    - [ ] Mandatories documented

  creative_guidance:
    - [ ] Tone and manner described
    - [ ] Reference examples provided
    - [ ] Restrictions clearly stated
    - [ ] Brand guidelines attached
```

### Quality Standards
```yaml
brief_quality_criteria:
  clarity:
    description: "Brief is easily understood"
    checks:
      - No jargon or ambiguous language
      - Single interpretation possible
      - All acronyms defined

  actionability:
    description: "Creative team can act on it"
    checks:
      - Clear deliverables specified
      - Adequate context provided
      - Inspiration/references included

  measurability:
    description: "Success can be evaluated"
    checks:
      - Specific KPIs defined
      - Targets are realistic
      - Measurement method identified

  alignment:
    description: "Stakeholders agree"
    checks:
      - All approvers signed off
      - No conflicting requirements
      - Priorities are clear
```

## Common Brief Challenges

### Problem: Scope Creep
```yaml
prevention_strategies:
  - Document all requirements upfront
  - Get formal sign-off on scope
  - Establish change request process
  - Track additions vs. original brief

change_request_process:
  steps:
    - Requester submits change form
    - Impact assessment (time, cost, quality)
    - Stakeholder approval required
    - Brief updated and re-signed
    - Timeline adjusted if needed
```

### Problem: Conflicting Stakeholders
```yaml
resolution_approach:
  step_1: "Identify all stakeholders and their concerns"
  step_2: "Facilitate alignment meeting"
  step_3: "Document agreed priorities"
  step_4: "Escalate unresolved conflicts"
  step_5: "Get single decision-maker to arbitrate"
```
