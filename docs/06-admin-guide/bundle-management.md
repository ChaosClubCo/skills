# Bundle Management

Bundles are curated subsets of skills packaged for specific platform use cases. The Skills Library maintains 18 bundles across 3 platforms (Gemini, Codex, Copilot). Claude does not use bundles; its skills are organized by category directly.

## How Bundles Are Generated

All bundles are generated by `populate_all.py` during the `--phase bundles` stage. The script:

1. Discovers all 508 master skills via `discover_all_skills()`
2. Parses each skill's SKILL.md for name, description, body, category, and line count
3. Applies dedup prefixes for slug collisions across categories
4. Scores skills using `score_skill()` for curated (size-limited) bundles
5. Writes platform-specific output files to the bundle directories

### Skill Scoring

The `score_skill()` function in `populate_all.py` ranks skills for curated bundles:

| Factor | Points |
|--------|--------|
| Body >= 50 lines | +2.0 |
| Body >= 100 lines | +2.0 |
| Body >= 200 lines | +1.0 |
| Description > 50 chars | +1.0 |
| Description contains "when" or "use" | +1.0 |
| Per heading (up to 10) | +0.3 each (max +3.0) |
| Per code block pair (up to 10) | +0.2 each (max +2.0) |
| Body < 20 lines | -3.0 |

Maximum theoretical score: approximately 12.0.

## Regenerating Bundles

To regenerate all bundles after updating master skills:

```bash
export PYTHONIOENCODING=utf-8
python populate_all.py --phase bundles
```

To regenerate everything (bundles + variants + cleanup):

```bash
python populate_all.py --phase all
```

Dry run to preview what would be generated:

```bash
python populate_all.py --dry-run --phase bundles
```

## Creating a New Bundle

To add a new bundle, you need to modify `populate_all.py`.

### Step 1: Define the Bundle Function

Add a new function in `populate_all.py` following the existing pattern. Example: creating a "data-science-25" bundle for Gemini with the top 25 data-related skills.

```python
def populate_gemini_data_science_25(skills: List[Dict]) -> int:
    """Populate Gemini data-science-25 bundle."""
    target = GEMINI_BUNDLES / "data-science-25"
    target.mkdir(parents=True, exist_ok=True)

    # Filter skills relevant to data science
    data_skills = [
        s for s in skills
        if any(kw in s["name"].lower() + " " + s["description"].lower()
               for kw in ["data", "analytics", "ml", "machine-learning", "statistics"])
    ]

    # Score and take top 25
    ranked = sorted(data_skills, key=score_skill, reverse=True)[:25]

    count = 0
    for skill in ranked:
        gem_json = build_gem_json(skill)
        out_file = target / f"{skill['slug']}.json"
        out_file.write_text(gem_json, encoding="utf-8")
        count += 1

    return count
```

### Step 2: Register the Bundle

Find the `populate_bundles()` function (or equivalent bundle orchestration section) in `populate_all.py` and add a call to the new function:

```python
count = populate_gemini_data_science_25(skills)
print(f"  data-science-25: {count} gems")
```

### Step 3: Test

```bash
python populate_all.py --phase bundles
find Gemini/GeminiSkills/bundles/data-science-25 -name "*.json" | wc -l
# Expected: 25
```

## Modifying an Existing Bundle

### Changing the Size

To change a bundle's size (e.g., expanding `studio-essential-30` to 40), find the corresponding function in `populate_all.py` and update the slice:

```python
# Before
ranked = sorted(skills, key=score_skill, reverse=True)[:30]

# After
ranked = sorted(skills, key=score_skill, reverse=True)[:40]
```

Also rename the directory if the name includes the count:

```python
# Before
target = GEMINI_BUNDLES / "studio-essential-30"

# After
target = GEMINI_BUNDLES / "studio-essential-40"
```

### Changing the Selection Criteria

To change which skills are included, modify the filter logic in the bundle function. For example, to add a category filter:

```python
# Before: all skills
ranked = sorted(skills, key=score_skill, reverse=True)[:50]

# After: only technical and ai-agents
filtered = [s for s in skills if s["category"] in {"technical", "ai-agents"}]
ranked = sorted(filtered, key=score_skill, reverse=True)[:50]
```

## Removing a Bundle

1. Delete the function from `populate_all.py`
2. Remove the function call from the bundle orchestration section
3. Delete the bundle directory:

```bash
rm -rf Gemini/GeminiSkills/bundles/deprecated-bundle
```

## Bundle Format Details

### Gemini Bundle Files

Each file is a Gem JSON object:

```json
{
  "name": "skill-name",
  "description": "Skill description...",
  "systemInstruction": "Full skill body...",
  "settings": {
    "temperature": 0.3,
    "model": "gemini-2.5-flash",
    "topP": 0.95,
    "topK": 40,
    "maxOutputTokens": 8192
  },
  "safetySettings": [...],
  "metadata": {
    "category": "technical",
    "complexity": "moderate",
    "version": "1.0.0",
    "source": "master-skills"
  }
}
```

### Codex Bundle Files

Responses API JSON with model, tools, and instructions.

### Copilot Bundle Files

Markdown files with custom instruction formatting, including `applyTo` file patterns and scope metadata in YAML frontmatter.
