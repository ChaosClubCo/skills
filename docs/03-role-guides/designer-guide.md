# Designer Guide

This guide curates the most valuable skills for UI/UX designers, visual designers, and design system leads. Skills are organized by design discipline so you can find the right tools for your current project, from wireframing through production handoff.

---

## UI and Visual Design

Core skills for creating interfaces and visual systems.

| Skill | Category | Description |
|-------|----------|-------------|
| `ui-design` | creative | User interface design principles, patterns, and best practices |
| `advanced-visual-design` | creative | Advanced visual design techniques including composition, hierarchy, and polish |
| `design-principles` | creative | Foundational design principles including balance, contrast, and alignment |
| `color-theory-application` | creative | Applying color theory to create effective palettes and accessible color systems |
| `typography-mastery` | creative | Typography selection, pairing, hierarchy, and responsive type scales |
| `ui-styling` | creative | CSS and styling techniques for implementing UI designs |
| `mobile-first-ui` | creative | Mobile-first design methodology and touch-optimized interface patterns |
| `premium-saas-design` | ai-agents | High-quality SaaS interface design with modern aesthetics and conventions |
| `aesthetic` | creative | Defining and maintaining consistent visual aesthetics across products |
| `canvas-design` | creative | Canvas-based design for interactive and data-rich interfaces |

### Suggested Workflow: Designing a New Feature

1. Establish visual direction with `design-principles` and `aesthetic`
2. Select colors with `color-theory-application`
3. Set typography with `typography-mastery`
4. Design the interface with `ui-design`
5. Refine with `advanced-visual-design` for polish

## UX and Research

Skills for understanding users and designing effective experiences.

| Skill | Category | Description |
|-------|----------|-------------|
| `ux-research` | ai-agents | Planning and conducting UX research studies |
| `user-research-methods` | creative | Qualitative and quantitative user research techniques |
| `design-research` | ai-agents | Discovery research methods including contextual inquiry and diary studies |
| `usability-testing` | technical | Conducting usability tests and synthesizing findings |
| `design-thinking` | technical | Design thinking methodology from empathy through prototyping |
| `information-architecture` | ai-agents | Organizing content structures, navigation, and taxonomies |
| `interaction-design` | ai-agents | Designing user interactions, transitions, and feedback patterns |
| `wireframing` | ai-agents | Rapid wireframing for concept exploration and stakeholder alignment |
| `ux-writing` | creative | Writing UI copy, microcopy, error messages, and onboarding text |
| `user-story-writing` | creative | Writing user stories and acceptance criteria from a design perspective |
| `accessibility-core` | creative | WCAG compliance, screen reader support, and inclusive design patterns |
| `accessibility-design` | ai-agents | Designing for accessibility across vision, motor, cognitive, and hearing needs |

### Suggested Workflow: Running a UX Project

1. Plan research with `ux-research` and `user-research-methods`
2. Organize findings into `information-architecture`
3. Explore solutions with `wireframing` and `design-thinking`
4. Design interactions with `interaction-design`
5. Write UI copy with `ux-writing`
6. Validate with `usability-testing`
7. Check accessibility with `accessibility-core`

## Design Systems

Skills for building and maintaining scalable design systems.

| Skill | Category | Description |
|-------|----------|-------------|
| `design-system-builder` | technical | Building comprehensive design systems with tokens, components, and docs |
| `design-system-foundation` | technical | Establishing design tokens, primitives, and foundational elements |
| `design-system-context` | ai-agents | Managing design system context for consistent cross-platform implementation |
| `ui-component-library` | technical | Creating and documenting reusable UI component libraries |
| `responsive-design-patterns` | creative | Responsive layout patterns including breakpoints, grids, and fluid design |
| `brand-systems` | creative | Building systematic brand identity frameworks |
| `brand-guidelines` | ai-agents | Creating and maintaining brand guideline documentation |
| `style-guides` | ai-agents | Developing style guides for visual and written consistency |
| `deliverable-templates` | creative | Creating reusable design deliverable templates |

### Suggested Workflow: Building a Design System

1. Define foundations with `design-system-foundation`
2. Establish brand with `brand-systems` and `brand-guidelines`
3. Build components with `ui-component-library`
4. Document in `style-guides`
5. Ensure responsiveness with `responsive-design-patterns`
6. Scale with `design-system-builder`
