# Role Guides

This section provides curated skill recommendations organized by professional role. Rather than browsing all 507 skills, you can jump directly to the guide that matches your job function and find the most relevant skills for your daily work.

## Available Role Guides

| Role | Guide | Key Focus Areas |
|------|-------|-----------------|
| Software Developer | [Developer Guide](developer-guide.md) | API development, frontend, DevOps, testing, architecture, code quality |
| Product Manager | [Product Manager Guide](product-manager-guide.md) | Strategy, discovery, launches, metrics, stakeholder management |
| Designer | [Designer Guide](designer-guide.md) | UI/UX, design systems, accessibility, visual design, motion |
| Data Analyst | [Data Analyst Guide](data-analyst-guide.md) | Analytics, data modeling, SQL, visualization, dashboards |
| DevOps / SRE | [DevOps Guide](devops-guide.md) | Kubernetes, Terraform, monitoring, incident response, CI/CD |
| Executive / Leadership | [Executive Guide](executive-guide.md) | Strategic planning, board reporting, growth, M&A, governance |
| Content Writer / Marketer | [Content Writer Guide](content-writer-guide.md) | SEO, content marketing, copywriting, social media, brand |

## How to Use These Guides

Each role guide includes:

- **Skill groupings** organized by subcategory relevant to that role
- **Brief descriptions** of what each skill does
- **Category tags** showing where each skill lives in the master library
- **Suggested workflows** combining multiple skills for common tasks

## Cross-Role Skills

Some skills appear in multiple role guides because they are broadly useful:

- **stakeholder-management** -- Relevant to PMs, executives, and senior developers
- **technical-writing** -- Useful for developers, PMs, and content writers
- **analytics-reporting** -- Applies to data analysts, PMs, and executives
- **design-system-builder** -- Shared between designers and frontend developers
- **incident-response** -- Important for DevOps engineers and engineering leadership

## Skill Naming Convention

All skill names use lowercase slugs with hyphens (e.g., `api-design-principles`). These slugs correspond directly to directory names in the master skills library under `_master-skills/{category}/{slug}/SKILL.md`.
