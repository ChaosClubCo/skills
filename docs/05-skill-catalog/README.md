# Skill Catalog

This directory contains the browsable catalog of all skills in the Skills Library.
The catalog is generated from the 507 master SKILL.md files in `_master-skills/`
and provides three views into the same data.

## Catalog Files

| File | Description |
|------|-------------|
| [full-catalog.md](full-catalog.md) | All skills in a single alphabetical table |
| [by-category.md](by-category.md) | Skills grouped by their 6 categories |
| [by-complexity.md](by-complexity.md) | Skills grouped by estimated complexity |

## Categories

The library spans 6 categories:

| Category | Count | Focus |
|----------|-------|-------|
| ai-agents | 232 | AI workflows, automation, MCP tools, LLM patterns |
| technical | 127 | Development, DevOps, testing, architecture |
| strategy | 59 | Business planning, analytics, compliance, growth |
| creative | 40 | Design, media production, visual arts, UX |
| operations | 24 | Process management, procurement, release workflows |
| industry | 25 | Sector-specific skills (healthcare, fintech, logistics, etc.) |

## Complexity Levels

Each skill is assigned an estimated complexity based on its description length,
section count, and slug structure:

- **Simple** -- Single-purpose skills with straightforward application
- **Moderate** -- Multi-step skills requiring some domain knowledge
- **Complex** -- Deep-domain skills with many processes and integration points

## Regenerating the Catalog

The catalog files are auto-generated by `_generate_catalog.py`. To regenerate
after adding or modifying skills:

```bash
python docs/05-skill-catalog/_generate_catalog.py
```

The script reads every `_master-skills/{category}/{slug}/SKILL.md` file,
extracts YAML frontmatter (name and description), counts markdown sections,
estimates complexity, and writes the three output files to this directory.

No external dependencies are required -- the script uses only the Python
standard library and parses YAML frontmatter with simple string matching.

## Skill File Format

Each master skill follows this structure:

```
_master-skills/{category}/{slug}/SKILL.md
```

With YAML frontmatter:

```yaml
---
name: skill-slug-name
description: Short description of what the skill does.
---
```

Followed by markdown body sections such as Overview, When to Use,
Core Processes, Tools and Templates, Metrics, Common Pitfalls, and
Integration Points.
