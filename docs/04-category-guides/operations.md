# Operations Category Guide

## Example Workflows

### Workflow 1: Automating a Procurement Process

1. **procurement-workflows** -- Map the current requisition-to-payment process
2. **business-process-automation** -- Identify automation opportunities in approval routing
3. **vendor-management** -- Establish vendor evaluation criteria and onboarding procedures
4. **compliance-monitoring** -- Set up automated policy checks for procurement thresholds
5. **operational-reporting** -- Build dashboards for spend analytics and cycle time tracking

### Workflow 2: Standing Up Incident Management

1. **incident-management-ops** -- Define triage severity levels, escalation paths, and communication protocols
2. **change-management-ops** -- Establish change advisory board and approval workflows
3. **service-level-agreements** -- Define SLAs for response and resolution times by severity
4. **slack-integration** -- Build Slack bots for incident alerts and status updates
5. **operational-reporting** -- Track MTTR, incident volume, and root cause categories

### Workflow 3: Launching a New Operational Team

1. **project-bootstrapper** -- Set up project structure, tooling, and documentation
2. **policy-development** -- Create team policies, procedures, and governance
3. **workflow-orchestration** -- Design cross-team workflows with dependencies
4. **capacity-planning** -- Model team capacity against demand forecasts
5. **creative-briefs** -- Standardize request intake with structured brief templates
6. **zapier-automation** -- Connect tools for automatic status updates and notifications
