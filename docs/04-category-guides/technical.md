## Example Workflows

### Workflow 1: Building a Production API Service

1. **api-design-principles** -- Define RESTful resource models, versioning, and error format
2. **fastapi-templates** or **nodejs-backend-patterns** -- Scaffold the project structure
3. **auth-implementation-patterns** -- Implement JWT/OAuth2 authentication
4. **postgresql** -- Design the database schema with proper indexing
5. **testing-strategies** -- Establish unit, integration, and E2E test coverage
6. **deployment-pipeline-design** -- Configure CI/CD with staging and production environments
7. **technical-documentation** -- Write API documentation and runbooks

### Workflow 2: Modernizing a Legacy Frontend

1. **react-modernization** -- Plan migration from class components to hooks
2. **modern-javascript-patterns** -- Adopt ES6+ features and functional patterns
3. **typescript-advanced-types** -- Add type safety incrementally
4. **design-system-foundation** -- Extract design tokens and build component library
5. **e2e-testing-patterns** -- Add Playwright tests before refactoring
6. **performance-optimization** -- Profile and optimize render performance
7. **code-review** -- Establish review guidelines for the migration

### Workflow 3: Setting Up Infrastructure from Scratch

1. **cloud-architecture** -- Select cloud provider and services
2. **terraform-iac-deployment** -- Define infrastructure as code
3. **github-actions-templates** -- Build CI/CD pipeline for infrastructure changes
4. **devops-practices** -- Establish deployment strategies and operational runbooks
5. **disaster-recovery** -- Plan backup procedures and recovery testing
6. **data-backup-recovery** -- Configure automated backups with RPO/RTO targets
7. **vercel-deployment** or **firebase-deployment** -- Configure application hosting
