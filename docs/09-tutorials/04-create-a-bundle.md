# Tutorial 04: Create a Bundle

Create a curated bundle of skills for a specific role or workflow. In this
tutorial you will build a "Frontend Developer Starter" bundle containing 15
skills relevant to frontend engineering, then generate platform-specific outputs
and deploy them.

---

## Prerequisites

- Completed [Tutorial 02](./02-run-the-full-pipeline.md) or familiarity with the
  pipeline scripts
- Python 3.10+, `PYTHONIOENCODING=utf-8` set on Windows
- The full pipeline has been run at least once (platform output directories exist)

---

## What Is a Bundle?

A bundle is a curated subset of skills packaged together for a specific audience.
The Skills Library ships 18 bundles across three platforms:

| Platform | Bundles | Examples |
|----------|---------|---------|
| Gemini | 7 | gems-full (508), studio-essential-30, vertex-enterprise (53) |
| Codex | 5 | responses-api-full (508), gpt-builder-50, agent-builder-20 |
| Copilot | 6 | workspace-essential-30, agent-skills-50, coding-agent-20 |

Bundles are generated by `populate_all.py` which reads from `_master-skills/`
and writes platform-formatted files into `bundles/` subdirectories.

## Step 2: Create the Bundle Directory Structure

Bundles are organized under each platform's `bundles/` directory. Create
directories for the Gemini, Codex, and Copilot variants:

```bash
$ mkdir -p Gemini/GeminiSkills/bundles/frontend-starter-15
$ mkdir -p Codex/CodexSkills/bundles/frontend-starter-15
$ mkdir -p GithubCopilot/CopilotSkills/bundles/frontend-starter-15
```

## Step 3: Add the Bundle to populate_all.py

Open `populate_all.py` in your editor. Find the section where named bundles are
defined (search for `# -- Copilot named bundles --` or the last bundle
definition before the function returns).

Add the following block after the existing Copilot bundle definitions (before the
`return counts` line in the `populate_bundles` function):

```python
    # ── Frontend Starter bundle (all 3 platforms) ──
    FRONTEND_STARTER_SLUGS = [
        "frontend-development",
        "react-modernization",
        "ui-component-library",
        "responsive-design-patterns",
        "modern-javascript-patterns",
        "typescript-advanced-types",
        "e2e-testing-patterns",
        "javascript-testing-patterns",
        "performance-optimization",
        "accessibility-core",
        "mobile-first-ui",
        "design-system-builder",
        "ui-styling",
        "frontend-design",
        "web-development",
    ]
    frontend_skills = [s for s in skills if s["slug"] in FRONTEND_STARTER_SLUGS]

    # Gemini variant
    target = GEMINI_BUNDLES / "frontend-starter-15"
    for s in frontend_skills:
        out = target / f"{s['slug']}.gem.json"
        if not dry_run:
            out.parent.mkdir(parents=True, exist_ok=True)
            out.write_text(build_gem_json(s), encoding='utf-8')
    counts["gemini/frontend-starter-15"] = len(frontend_skills)

    # Codex variant
    target = CODEX_BUNDLES / "frontend-starter-15"
    for s in frontend_skills:
        out = target / f"{s['slug']}.response.json"
        if not dry_run:
            out.parent.mkdir(parents=True, exist_ok=True)
            out.write_text(build_codex_response_json(s), encoding='utf-8')
    counts["codex/frontend-starter-15"] = len(frontend_skills)

    # Copilot variant
    target = COPILOT_BUNDLES / "frontend-starter-15"
    for s in frontend_skills:
        out = target / f"{s['slug']}.md"
        if not dry_run:
            out.parent.mkdir(parents=True, exist_ok=True)
            out.write_text(build_copilot_instruction(s), encoding='utf-8')
    counts["copilot/frontend-starter-15"] = len(frontend_skills)
```

## Step 4: Generate the Bundle

Run `populate_all.py` with the bundles phase:

```

  codex/frontend-starter-15: 15
  copilot/frontend-starter-15: 15
```

## Step 6: Deploy the Bundle

To deploy only the bundle (not the entire skill set), copy just the bundle
directory.

**Gemini:**

```bash
$ cp -r Gemini/GeminiSkills/bundles/frontend-starter-15 \
       ~/.gemini/skills/frontend-starter-15
```

**Copilot:**

```bash
$ cp -r GithubCopilot/CopilotSkills/bundles/frontend-starter-15 \
       .github/copilot-instructions/frontend-starter-15
```

For full deployment, use the CLI variant copy commands as described in
[Tutorial 01](./01-create-your-first-skill.md#step-7-deploy-to-your-ai-assistant).

## Bundle Design Guidelines

### Choosing Skills for a Bundle

1. **Start with a role or workflow.** Define who the bundle is for (frontend
   developer, data engineer, startup founder) or what workflow it supports
   (code review, CI/CD setup, API development).

2. **Aim for 15-50 skills.** Fewer than 10 is too narrow to be useful. More
   than 50 may overwhelm the AI context window.

3. **Mix skill types.** Include foundational skills (patterns, principles),
   practical skills (implementation guides), and quality skills (testing,
   review checklists).

4. **Check for dependencies.** If skill A references skill B in its Integration
   Points section, include skill B in the bundle.

### Naming Conventions

Bundle directory names follow this pattern:

```
{descriptive-name}-{count}
```

Examples: `studio-essential-30`, `coding-agent-20`, `vertex-enterprise`.

For bundles without a fixed count (like `by-category` or `gems-full`), omit
the count suffix.

### Updating an Existing Bundle

To add or remove skills from a bundle, edit the slug list in `populate_all.py`
and re-run:

```bash
$ python populate_all.py --phase bundles
```

The entire bundle directory is regenerated each time, so removed skills will
no longer appear.
