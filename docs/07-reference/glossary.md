# Glossary

Definitions of key terms used throughout the Skills Library documentation and codebase.

---

## A

### Adapter

A JavaScript module in `UniversalAdapters/` that transforms a master skill into a platform-specific format at runtime. There are 8 adapters: `claude-cli`, `claude-desktop`, `claude-web`, `codex-cli`, `copilot-cli`, `gemini-cli`, `github-repo`, and `index`. Each adapter reads a SKILL.md and outputs the target platform's format.

### Agent Builder

An OpenAI format for creating autonomous agent configurations. Agent Builder output includes the skill body prefixed with an agent loop instruction set (`AGENT_LOOP_PREFIX` in `lib/config.py`) that directs the agent to break tasks into verifiable steps, use tools proactively, and provide structured output. Generated by `convert_to_codex_responses.py`.

---

## B

### Bundle

A curated subset of skills packaged together for a specific platform use case. Bundles are generated by `populate_all.py` and stored in `{Platform}/bundles/` directories. There are 18 bundles across 3 platforms (Gemini, Codex, Copilot). Bundle sizes range from 15 to 508 skills. See [Bundle Management](../06-admin-guide/bundle-management.md).

---

## C

### Category

One of six top-level classifications for skills. Categories determine directory placement, temperature settings, tool assignments, and file pattern mappings. The six categories are: `ai-agents`, `technical`, `strategy`, `creative`, `operations`, `industry`. Defined in `CATEGORIES` in `lib/config.py`.

### Collision Group

A set of skills in different categories that share the same directory name (slug). There are 5 known collision groups. Skills in collision groups receive a dedup prefix during conversion. See [Slug Collision Reference](slug-collision-reference.md).

### Complexity

A computed classification of a skill's sophistication level: `simple`, `moderate`, or `complex`. Determined by `estimate_complexity()` in `lib/platform_tuning.py` based on line count, section count, code blocks, templates, and framework references. Complexity determines which model tier is assigned to a skill.

### Converter

A script that transforms master SKILL.md files into a platform-specific format. The five converters are: `convert_to_gems.py` (Gemini), `convert-to-copilot.js` (Copilot), `convert_to_codex_responses.py` (Codex), `convert_to_cli_skills.py` (CLI variants), and the Claude converter built into `sync-skills.py`.

---

## D

### Declarative Agent

A Gemini agent configuration that pairs a GEMINI.md instruction file with a JSON metadata file. Generated for the `GeminiSkills-Agents` output in `Gemini/GeminiSkills-Agents/.gemini/agents/{slug}/`. Each agent has tool assignments from `GEMINI_AGENT_TOOLS` in `lib/config.py`.

### Dedup Prefix

A disambiguation pattern applied to skills in collision groups. When two skills in different categories share the same slug, both receive a `{category}--{slug}` prefix in platform outputs. For example, `ai-agents--packaging-design` and `creative--packaging-design`. Applied by `discover_all_skills()` in `populate_all.py`.

---

## E

### Enricher

The metadata enrichment module (`lib/metadata_enricher.py`) that adds computed fields to skill data during conversion. Fields include: tags, complexity, token estimates, related skills, platform capabilities, section count, and code block count. Entry point: `enrich_skill()`.

---

## F

### Frontmatter

The YAML metadata block at the top of every SKILL.md file, delimited by `---` markers. Contains at minimum `name` (the skill's slug) and `description` (a trigger-rich text explaining what the skill does and when to use it). Frontmatter is parsed by every converter and validator in the pipeline.

---

## G

### Gem

A Gemini-specific skill format stored as a JSON file. Contains the skill's system instruction, model settings (temperature, topP, topK, maxOutputTokens), safety settings, and metadata. Generated by `convert_to_gems.py` and stored in `Gemini/GeminiSkills/gems/{category}/{slug}.json`.

### GPT Builder

An OpenAI format for creating custom GPT configurations. Includes capability flags (web_browsing, code_interpreter, image_generation, file_upload) mapped per category in `CAPABILITIES_MAP` in `lib/config.py`. Generated by `convert_to_codex_responses.py`.

### Grounding

A Gemini feature that enables the model to verify facts against Google Search results. Applied to skills in `GROUNDING_CATEGORIES` (strategy, industry, operations) by prefixing the system instruction with `GROUNDING_PREFIX` from `lib/config.py`.

---

## M

### Master Skill

The authoritative version of a skill, stored in `_master-skills/{category}/{slug}/SKILL.md`. All platform outputs are generated from master skills. Edits should only be made to master skills, never to generated outputs.

---

## P

### Pipeline

The end-to-end process that transforms master skills into platform outputs. Consists of 5 stages: quality fixes (3-pass), sync/convert, bundle population, variant fixes, and cleanup. Orchestrated by running `fix_skills.py`, `fix_skills_structure.py`, `fix_skills_pass3.py`, `sync-skills.py`, and `populate_all.py` in sequence.

### Platform Output

A generated file in a platform-specific format derived from a master skill. Each master skill produces outputs for Claude (SKILL.md), Gemini (JSON gem), Codex (JSON + TXT), and Copilot (Markdown). Platform outputs are stored in separate directory trees per platform.

---

## R

### Responses API

The OpenAI Responses API format, the primary Codex output format. Each skill is converted to a JSON object with `model`, `instructions`, `tools`, and `metadata` fields. Generated by `convert_to_codex_responses.py` and stored in `Codex/CodexSkills/`.

---

## S

### Safety Settings

Gemini-specific configuration that controls content filtering thresholds. The pipeline applies `BLOCK_ONLY_HIGH` threshold to five harm categories: harassment, hate speech, sexually explicit content, dangerous content, and civic integrity. Defined in `GEMINI_HARM_CATEGORIES` and `GEMINI_SAFETY_SETTINGS` in `lib/config.py`.

### Slug

The URL-safe, lowercase, hyphenated identifier for a skill. Derived from the skill's directory name (not from `slugify(frontmatter_name)`). Must match the pattern `[a-z0-9]+(-[a-z0-9]+)*`. Examples: `api-development`, `brand-guidelines`, `workflow-optimizer`.

### System Prompt

A plain-text version of a skill's instructions, generated as a `.txt` file alongside the JSON outputs in `Codex/CodexSkills/`. Intended for direct use as a system message in OpenAI API calls without the Responses API wrapper.

---

## T

### Trigger Verb

An action verb included in a skill's description to make it discoverable when users search or when AI systems match skills to tasks. The pipeline enforces that every description contains at least one trigger verb. The full list of 44 trigger verbs is defined in `TRIGGER_VERBS` in both `lib/config.py` and `lib/skill_validator.py`. Examples: create, analyze, build, manage, optimize, generate, design, deploy.

---

## V

### Validator

The validation module (`lib/skill_validator.py`) that checks skills against quality standards and produces a 0-100 quality score. The `SkillValidator` class runs five check categories: frontmatter, body, structure, description quality, and cross-references. Entry point: `validate()` for single skills, `validate_all()` for batch.

### Variant

A platform-specific alternative output format. Four thin variants are maintained: Copilot Frontier (coding agent), Gemini Agents (declarative agents), Claude Desktop (`.claude-plugin` format), and Claude Web (`project-instructions.md` format). Generated by `populate_all.py --phase variants`.
