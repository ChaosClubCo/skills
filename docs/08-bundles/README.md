# Bundles

Bundles are curated, deployment-ready subsets of the Skills Library. Instead of deploying all 508 skills to a platform, bundles let you deploy a focused collection tailored to a specific persona, workflow, or deployment context.

---

## Why Bundles Exist

The full Skills Library contains 508 skills across 6 categories. That breadth is valuable for coverage, but most users do not need all 508 at once. A front-end developer working in VS Code needs different skills than an enterprise architect deploying to Vertex AI.

Bundles solve three problems:

1. **Relevance** -- Deploy only the skills that match a user's role, project, or platform context.
2. **Performance** -- Smaller bundles load faster and reduce noise in AI completions and suggestions.
3. **Simplicity** -- A curated bundle requires zero configuration. Copy the directory and start working.

---

## Design Philosophy

Bundles follow two organizing principles:

**Persona-targeted bundles** contain skills selected for a specific role or workflow. The `coding-agent-20` bundle, for example, contains the 20 highest-scoring skills suitable for autonomous coding agents. The `studio-creative-15` bundle targets designers and content creators using Google AI Studio.

**Deployment-context bundles** match a specific platform feature or infrastructure target. The `responses-api-full` bundle contains all 508 skills formatted for the OpenAI Responses API. The `workspace-by-stack` bundle organizes skills into 8 technology-specific subdirectories for Copilot workspaces.

Every bundle is generated by `populate_all.py` using the `score_skill()` ranking function. Ranked bundles (those with a number in the name, like `agent-skills-50`) take the top N skills by score. Full bundles (`gems-full`, `by-category`, `responses-api-full`) include all 508 skills with platform-specific formatting.

---

## When to Use Bundles vs. Individual Skills

| Scenario | Recommendation |
|----------|----------------|
| Exploring the library for the first time | Deploy a full bundle (gems-full, responses-api-full, or by-category) |
| Working in a specific role or domain | Choose a themed bundle (vertex-enterprise, coding-agent-20) |
| Need just one or two skills | Copy individual SKILL.md files directly from `_master-skills/` |
| Building a custom deployment for your team | Create a custom bundle (see [Creating Bundles](./creating-bundles.md)) |

## Quick Deploy

Copy a bundle directory to your platform's expected location:

```bash
# Gemini CLI -- deploy all gems
cp -r GeminiSkills-CLI/.gemini ~/.gemini

# Gemini -- deploy essential 30 only
cp -r GeminiSkills/bundles/studio-essential-30/* ~/.gemini/gems/

# Copilot CLI -- deploy full instruction set
cp -r CopilotSkills-CLI/.github .github

# Copilot -- deploy coding agent bundle
cp -r CopilotSkills/bundles/coding-agent-20/ .github/skills/

# Codex CLI -- deploy full CLI skill set
cp -r CodexSkills-CLI/.codex ~/.codex

# Codex -- deploy Responses API bundle
cp -r CodexSkills/bundles/responses-api-full/ ~/.codex/skills/
```

## Regenerating Bundles

Bundles are generated by the pipeline, not maintained by hand:

```bash
PYTHONIOENCODING=utf-8 python populate_all.py --phase bundles
```

To preview without writing files:

```bash
PYTHONIOENCODING=utf-8 python populate_all.py --phase bundles --dry-run
```

The script discovers all 508 master skills, scores them with `score_skill()`, sorts by score, and populates each bundle directory according to its selection criteria. The full bundle phase generates approximately 3,000 files across all 18 bundles.
