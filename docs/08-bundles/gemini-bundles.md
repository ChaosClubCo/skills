# Gemini Bundles

This guide covers the 7 bundles available for the Gemini platform. Each bundle is located under `GeminiSkills/bundles/` and contains Gem JSON files ready for deployment to Gemini CLI, AI Studio, Vertex AI, or Project IDX.

---

## Overview

| Bundle | Skills | Format | Target |
|--------|-------:|--------|--------|
| [gems-full](#gems-full) | 508 | Gem JSON by category | Complete library |
| [by-category](#by-category) | 508 | Gem JSON by category | Browsable full library |
| [vertex-enterprise](#vertex-enterprise) | 53 | Gem JSON | Enterprise/cloud teams |
| [studio-essential-30](#studio-essential-30) | 30 | Gem JSON | AI Studio starter set |
| [agent-chains-20](#agent-chains-20) | 20 | Gem JSON | Agent orchestration |
| [idx-workspace](#idx-workspace) | 20 | Gem JSON | IDX cloud IDE |
| [studio-creative-15](#studio-creative-15) | 15 | Gem JSON | Creative workflows |

---

## Gem JSON Format

Every Gemini bundle file is a `.gem.json` file generated by `convert_to_gems.py` (called internally by `populate_all.py`). Each file includes:

- `name` -- The skill display name
- `description` -- Short description from frontmatter
- `instructions` -- Full skill body converted to Gem instruction format
- `model` -- Recommended Gemini model, assigned dynamically by category and complexity
- `temperature` -- Tuned per category using `CATEGORY_TEMPERATURES` from `lib/config.py`
- `safety_settings` -- Platform-appropriate safety configuration from `GEMINI_SAFETY_SETTINGS`
- `grounding` -- Search grounding settings where applicable

Category-level temperature defaults:

| Category | Temperature |
|----------|------------|
| technical | 0.2 |
| operations | 0.3 |
| industry | 0.3 |
| strategy | 0.5 |
| ai-agents | 0.4 |
| creative | 0.7 |

## gems-full

**Path:** `GeminiSkills/bundles/gems-full/`
**Skills:** 508
**Format:** One `.gem.json` file per skill, organized by category subdirectories

### Contents

Every skill in the master library converted to Gemini Gem JSON format. Files are placed in `{category}/{slug}.gem.json` subdirectories.

### Directory Structure

```
GeminiSkills/bundles/gems-full/
  ai-agents/
    autonomous-agent-design.gem.json
    multi-agent-orchestration.gem.json
    ...
    (232 files)
  technical/
    api-design-principles.gem.json
    automated-testing-strategy.gem.json
    ...
    (127 files)
  strategy/
    (59 files)
  creative/
    (40 files)
  operations/
    (25 files)
  industry/
    (25 files)
```

### Deployment

```bash
# Deploy all gems to Gemini CLI configuration
cp -r GeminiSkills/bundles/gems-full/* ~/.gemini/gems/

# Or deploy the entire CLI skill set (includes config)
cp -r GeminiSkills-CLI/.gemini ~/.gemini
```

### When to Choose

Use gems-full when you want the entire library available and are comfortable browsing 508 files. Best for power users who know what they need or who want to search across all skills.

## by-category

**Path:** `GeminiSkills/bundles/by-category/`
**Skills:** 508
**Format:** Gem JSON organized into 6 category subdirectories

### Contents

Same Gem JSON files as gems-full, organized into subdirectories matching the 6 master categories. The difference from gems-full is structural: by-category exists as a separate bundle for teams that want to deploy individual categories independently.

### Directory Structure

```
GeminiSkills/bundles/by-category/
  ai-agents/
    autonomous-agent-design.gem.json
    ...
    (232 files)
  technical/
    api-design-principles.gem.json
    ...
    (127 files)
  strategy/
    ...
    (59 files)
  creative/
    ...
    (40 files)
  operations/
    ...
    (25 files)
  industry/
    ...
    (25 files)
```

### Deployment

```bash
# Deploy all categories
cp -r GeminiSkills/bundles/by-category/ ~/.gemini/gems/

# Deploy a single category
cp -r GeminiSkills/bundles/by-category/technical/ ~/.gemini/gems/technical/

# Deploy two categories for a compliance team
cp -r GeminiSkills/bundles/by-category/operations/ ~/.gemini/gems/operations/
cp -r GeminiSkills/bundles/by-category/industry/ ~/.gemini/gems/industry/
```

### When to Choose

Use by-category when you prefer organized browsing or want to deploy only specific categories. This is the recommended bundle for most users because the category structure makes it easy to find relevant skills and deploy subsets.

## vertex-enterprise

**Path:** `GeminiSkills/bundles/vertex-enterprise/`
**Skills:** 53
**Format:** Gem JSON

### Contents

A curated set of 53 skills focused on enterprise concerns. The selection criteria pull from three sources:

1. All 25 skills from the `operations` category
2. All 25 skills from the `industry` category
3. Skills from `strategy` whose name contains: enterprise, compliance, governance, risk, or audit

All selected skills are sorted by `score_skill()`.

Representative skills include:

- enterprise-architecture
- cloud-migration-strategy
- compliance-automation
- infrastructure-as-code
- kubernetes-orchestration
- data-governance
- api-gateway-design
- disaster-recovery-planning
- cost-optimization
- identity-access-management
- hipaa-compliance
- pci-compliance

### Deployment

```bash
# Deploy vertex enterprise bundle
cp -r GeminiSkills/bundles/vertex-enterprise/ ~/.gemini/gems/

# Integrate with Vertex AI agent configuration
cp -r GeminiSkills/bundles/vertex-enterprise/ /path/to/vertex-agent/gems/
```

### Configuration Tips

When deploying to Vertex AI, consider adjusting the temperature settings in the Gem JSON files. Enterprise contexts typically benefit from lower temperatures (0.1-0.3) for more deterministic outputs. The default category temperatures are already conservative for operations (0.3) and industry (0.3).

For production Vertex AI deployments, you may also want to:

- Enable stricter safety settings
- Set grounding to use your organization's internal knowledge base
- Pin a specific model version rather than using the latest

### When to Choose

Use vertex-enterprise when deploying Gemini in a corporate or cloud-platform context. This bundle skips creative, content, and general-purpose skills in favor of architecture, operations, and governance expertise. It is the Gemini counterpart to the Codex `enterprise-assistants` bundle.

## studio-essential-30

**Path:** `GeminiSkills/bundles/studio-essential-30/`
**Skills:** 30
**Format:** Gem JSON

### Contents

The 30 highest-scoring skills across all categories, selected by `score_skill()`. This produces a balanced set that covers the most commonly needed capabilities without loading the full library.

Representative skills include:

- code-review-checklist
- api-design-principles
- automated-testing-strategy
- technical-writing
- project-planning
- data-analysis
- debugging-strategies
- performance-optimization
- documentation-generation
- prompt-engineering

### Deployment

```bash
# Deploy essential 30 to AI Studio
cp -r GeminiSkills/bundles/studio-essential-30/ ~/.gemini/gems/
```

### Configuration Tips

The essential-30 bundle is designed to work well with default settings. Since it pulls from multiple categories, the temperature settings will vary per file (0.2 for technical skills, 0.7 for creative ones). This is intentional -- each skill retains the temperature appropriate to its domain.

### When to Choose

Use studio-essential-30 as your starting point if you are new to the library or want a lean deployment. You can always add individual skills from the full library later. This is the recommended bundle for personal AI Studio workspaces.

## agent-chains-20

**Path:** `GeminiSkills/bundles/agent-chains-20/`
**Skills:** 20
**Format:** Gem JSON

### Contents

The top 20 skills from the `ai-agents` category, ranked by `score_skill()`. These skills cover agent design patterns, tool use, chain-of-thought reasoning, memory management, and inter-agent communication.

Representative skills include:

- autonomous-agent-design
- multi-agent-orchestration
- agent-memory-management
- tool-use-optimization
- chain-of-thought-prompting
- agent-evaluation-framework
- retrieval-augmented-generation
- function-calling-patterns
- agent-safety-guardrails
- workflow-automation

### Deployment

```bash
# Deploy agent chain skills
cp -r GeminiSkills/bundles/agent-chains-20/ ~/.gemini/gems/
```

### Configuration Tips

Agent workflows often benefit from slightly higher temperatures (0.3-0.5) to encourage exploration, but the defaults (0.4 for ai-agents) are a reasonable starting point. If your agents need to be more deterministic, reduce the temperature in each Gem JSON to 0.2.

For multi-step agent chains, consider loading these Gems alongside a few technical skills (like `api-design-principles` or `error-handling-patterns`) from the full library to cover implementation details.

### When to Choose

Use agent-chains-20 when building multi-agent systems, LLM orchestration pipelines, or autonomous workflows. This bundle provides focused expertise for agent developers without the noise of unrelated skills.

## idx-workspace

**Path:** `GeminiSkills/bundles/idx-workspace/`
**Skills:** 20
**Format:** Gem JSON

### Contents

The top 20 skills from the `technical` category, ranked by `score_skill()`. Covers the core development loop: writing code, debugging, testing, documenting, deploying, and reviewing.

Representative skills include:

- code-generation
- debugging-strategies
- automated-testing-strategy
- code-review-checklist
- refactoring-patterns
- documentation-generation
- git-workflow-management
- ci-cd-pipeline-design
- containerization
- terminal-productivity

### Deployment

```bash
# Deploy to IDX workspace
cp -r GeminiSkills/bundles/idx-workspace/ ~/.gemini/gems/

# Place in IDX-specific project configuration
cp -r GeminiSkills/bundles/idx-workspace/ ~/idx-workspace/.gemini/gems/
```

### Configuration Tips

IDX workspaces have Gemini integration built in. When deploying this bundle, place the Gem files where IDX can discover them. The technical skills use a temperature of 0.2 by default, which is appropriate for code-focused work.

If your IDX project is primarily in one language, consider supplementing this bundle with language-specific skills from the full library.

### When to Choose

Use idx-workspace when working in Google Project IDX and you want a focused set of development skills without enterprise, creative, or strategy skills. This is the most compact technical bundle.

## Regenerating Gemini Bundles

All Gemini bundles are generated by `populate_all.py` during the pipeline's bundle phase:

```bash
PYTHONIOENCODING=utf-8 python populate_all.py --phase bundles
```

This regenerates all 18 bundles across all platforms. There is no flag to regenerate only Gemini bundles; the bundle phase runs all platforms together. To preview without writing:

```bash
PYTHONIOENCODING=utf-8 python populate_all.py --phase bundles --dry-run
```

The output shows the file count for each bundle:

